# Diff map

This action generates a diagram of the files changes in a pull request or push.

## Example usage

```yaml
uses: cundejo/diff-map@v1.1
```

## Publishing a new version

```shell
git tag -a -m "MESSAGE" v1.1
git push --follow-tags
```
